# 自动化脚本录制系统 (AutoStoryTranscribe)

## 概述

AutoStoryTranscribe 是一个用于原神的自动化脚本录制系统，可以生成地图追踪路径（.json）和任务流程脚本（process.json）。

## 主要功能

### 1. 基础录制功能
- 自动录制玩家移动路径
- 生成标准的地图追踪文件格式
- 支持路径优化和移动模式自动识别

### 2. 按键监听式操作系统
- 依赖按键监听：无需像原先一样打开界面

#### 对话功能
- **功能**：
  - 从游戏界面OCR提取当前任务描述中的NPC名称
  - 自动返回主界面并选择该NPC
  - 在地图追踪文件中保存路径
  - 在process.json中生成 `对话 [NPC名称]` 指令

#### 战斗功能  
- **功能**：
  - 在当前坐标生成一个 `type` 为 "fight" 的路径点
  - 标记为不可优化 (`optimize: false`)
  - 执行 `await dispatcher.RunTask("AutoFight")`

#### 暂停/恢复功能
- **首次按下（暂停）**：
  - 停止录制当前位置
  - 保存当前地图追踪文件
  - 在process.json中追加 `地图追踪文件.json` 和 `暂停` 指令
- **再次按下（恢复）**：
  - 创建新的地图追踪文件继续录制新路径
  - process.json中的后续指令将与新路径文件关联

#### 保存路径功能 (新增)
- **功能**：
  - 保存当前录制的路径到地图追踪文件
  - 在process.json中追加地图追踪文件指令
  - 自动开始新的路径文件继续录制
  - 允许用户手动分段保存路径

#### 结束录制功能
- **功能**：
  - 保存最后的路径文件
  - 在process.json中追加 `任务完成` 指令
  - 停止所有录制，保存文件

### 3. 安全机制

#### 最大录制次数限制 (新增)
- **功能**：防止无限循环导致系统资源占用
- **默认值**：72000次循环（约18小时，每0.9秒一次）
- **达到限制时**：
  - 自动停止录制
  - 如果有未保存的路径，会自动保存
  - 在process.json中添加 `录制超时自动结束` 标记
- **进度显示**：每1000次循环输出一次进度信息

### 4. 特殊情景处理

#### 情景A：录制中进入剧情 (ELEMENT_STATE.Story)
- 立即停止当前路径录制
- 在当前地图追踪文件的最后一个路径点添加：
  ```json
  {
    "type": "combat_script",
    "action_params": "w(5)"
  }
  ```
- 在process.json中添加 `等待返回主界面` 指令

#### 情景B：录制中发生长距离传送
- 检测坐标剧烈变化（超出配置阈值，默认100单位）判定为传送
- 在上一个路径点之后，新坐标点之前，插入特殊路径点：
  ```json
  {
    "x": [传送后X坐标],
    "y": [传送后Y坐标],
    "type": "teleport",
    "move_mode": "walk",
    "action_params": "",
    "optimize": false
  }
  ```

## 配置说明

### settings.json 配置项

| 配置项 | 类型 | 默认值 | 说明 |
|-------|------|--------|------|
| questName | 文本 | "默认" | 录制的任务名称，用于生成文件名 |
| questLocation | 文本 | "蒙德城" | 任务所在地区，用于文件夹组织 |
| trackNumber | 文本 | "1" | 任务追踪编号，用于区分同一任务的不同追踪 |
| runMode | 选择 | "录制模式" | 选择录制新任务还是运行已有任务 |
| keyDialogue | 选择 | "教程G" | 对话触发界面 |
| keyFight | 选择 | "好友O" | 战斗触发界面 |
| keyPause | 选择 | "联机F2" | 暂停/恢复触发界面 |
| keySave | 选择 | "纪行F4" | 保存路径触发界面 |
| keyEnd | 选择 | "任务J" | 结束录制触发界面 |
| keyExit | 选择 | "角色C" | 退出录制触发界面 |
| start | 选择 | "传送点" | 起始点类型：传送点或当前位置 |
| maxRecordingCycles | 文本 | "72000" | 最大录制循环次数限制 |
| strategyScript | 文本 | "w(10)" | 进入剧情时执行的等待脚本 |
| teleportThreshold | 文本 | "100" | 传送检测距离阈值 |
| enableTransmissionDetection | 复选框 | true | 是否启用传送检测功能 |
| author | 文本 | "" | 录制文件的作者信息 |
| description | 文本 | "" | 录制任务的描述信息 |
| version | 文本 | "1.0" | 录制文件的版本号 |

## 输出文件格式

### 地图追踪文件 (.json)
标准的BGI地图追踪文件格式，包含：
- 路径点数组
- 特殊点包含 `type`, `action_params`, `optimize` 等字段
- 支持战斗点、传送点、脚本执行点等特殊类型

### 流程脚本文件 (process.json)
使用文本格式（非JSON数组），结构示例：
```
任务描述：与纳西妲对话
地图追踪 path1.json
对话 纳西妲
地图追踪 path2.json
暂停
地图追踪 path3.json
任务完成
```

## 使用流程

### 1. 准备工作
1. 在BGI中加载 AutoStoryTranscribe 脚本
2. 配置 settings.json 中的各项参数
3. 确保游戏处于主界面状态

### 2. 开始录制
1. 运行脚本，系统开始录制玩家移动轨迹
2. 系统会自动检测界面状态变化
3. 根据需要按下对应按键触发功能

### 3. 录制过程中的操作

#### 触发对话
1. 移动到需要对话的NPC附近
2. 按下设置的按键
3. 系统自动识别任务中的NPC名称并生成对话指令

#### 触发战斗
1. 移动到战斗位置
2. 按下设置的按键
3. 系统在当前位置标记战斗点并启动自动战斗

#### 暂停录制
1. 按下设置的按键
2. 系统暂停录制并保存当前路径
3. 再次打开相同界面可恢复录制

### 4. 文件输出
录制完成后，系统会在以下位置生成文件：
```
assets/[任务地区]/[任务名称]/
├── [任务名称]-[编号]-path1.json    # 地图追踪文件1
├── [任务名称]-[编号]-path2.json    # 地图追踪文件2（如果有）
├── ...
└── process.json                     # 任务流程脚本
```

## 技术特点

### 智能NPC名称提取
- 使用OCR技术从任务描述中提取NPC名称
- 支持多种任务描述模式匹配
- 自动处理中文NPC名称识别

### 路径优化算法
- 使用RDP算法简化路径点
- 保留飞行、游泳、攀爬等关键状态转换点
- 自动识别移动模式并优化路径

### 传送检测机制
- 自动检测长距离位置变化
- 在路径中插入传送标记点
- 可配置检测阈值以适应不同场景

## 故障排除

### 常见问题

1. **界面检测不响应**
   - 确保在游戏主界面状态下使用
   - 检查界面配置是否正确
   - 尝试重新配置触发界面

2. **NPC名称识别失败**
   - 确保任务描述清晰可见
   - 检查OCR识别区域设置
   - 手动在process.json中补充NPC名称

3. **路径录制异常**
   - 检查是否在主界面状态
   - 确认小地图可见
   - 重启脚本重新开始录制

4. **传送检测误判**
   - 调整teleportThreshold阈值
   - 或禁用传送检测功能

5. **脚本启动时出现"Cannot access 'UIStateMonitor' before initialization"错误**
   - 这通常是JavaScript引擎的变量初始化顺序问题
   - 确保使用最新版本的脚本文件
   - 如果问题持续，尝试重新加载脚本

### 调试模式
在脚本运行时，日志会详细记录：
- 界面状态变化
- 坐标获取情况
- 文件保存状态
- NPC名称识别结果

## 注意事项

1. **使用环境**
   - 需要在BGI环境下运行
   - 确保游戏分辨率为16:9比例
   - 建议使用1920x1080分辨率以获得最佳识别效果

2. **录制质量**
   - 保持稳定的网络连接
   - 避免频繁的界面切换
   - 录制过程中保持游戏窗口为焦点状态

3. **文件管理**
   - 定期整理生成的追踪文件
   - 备份重要的录制结果
   - 注意文件命名规范
